JobBot — AI Email Automation Experiment

System Architecture · AI Automation · Cost Optimization

Overview

JobBot is a private, self-initiated experiment designed to solve a very practical problem: the time and cognitive overhead of handling dozens of recruiter emails daily, each requiring a tailored response and a customized resume.

The goal was not to design a polished consumer-facing product, but to prove that a powerful AI-driven automation system could be built cheaply, quickly, and almost entirely on top of free, existing tools. This project serves as both a personal productivity system and an MVP for a potential future commercial product.

Problem

Recruiter outreach follows a predictable but time-consuming pattern:

  • Incoming emails vary in quality and relevance
  • Each legitimate opportunity requires:
    • Careful reading of the job description
    • A personalized reply
    • A resume tailored to the role
  • Context switching and repetition significantly slow down the job search

The cost was not just time, but mental energy.

Solution

JobBot automates the entire workflow:

  • Detects and classifies recruiter emails
  • Extracts job context and requirements
  • Generates:
    • A personalized email response
    • A role-specific resume
  • Tracks activity and sends daily summaries

All while keeping infrastructure costs close to zero.

Key Principles
  • Almost no cost: only AI API usage is paid
  • Built on free tools: Google ecosystem
  • Simple, transparent architecture
  • Easy to replicate by others
  • Non-technical friendly operation
High-Level Flow
  1. Recruiter email arrives in Gmail
  2. Email is filtered and parsed automatically
  3. Job description and context are extracted
  4. AI generates:
    • Custom reply email
    • Tailored resume
  5. Draft is prepared and sent
  6. Activity is logged
  7. Daily summary email is sent to the user
System Architecture (Plain English)

JobBot runs entirely on top of standard Google tools:

Core Components

Gmail
Receives recruiter emails and sends replies

Google Sheets
Acts as the admin interface:

  • Logs all activity
  • Stores configuration and rules
  • Tracks recruiters, jobs, responses, and outcomes

Google Apps Script
The "brain" of the system:

  • Watches Gmail
  • Processes emails
  • Calls AI APIs
  • Writes data to Sheets
  • Sends reports

AI API (GPT)
Used only when needed to:

  • Analyze job descriptions
  • Generate resumes
  • Write personalized responses

Why This Architecture

This setup was chosen deliberately:

  • No servers to maintain
  • No databases to manage
  • No custom UI required
  • No vendor lock-in beyond the AI API

Anyone with:

  • A Google account
  • Basic scripting knowledge

can copy, modify, and deploy the system.

Cost Control Strategy
  • AI calls are only triggered for relevant emails
  • No embeddings or vector databases running continuously
  • No background processing unless needed
  • Google infrastructure handles scheduling and execution for free

Result: API costs remain minimal and predictable.

Admin & Monitoring

There is no custom dashboard.

Instead:

  • Google Sheets serves as the admin panel
  • Every action is logged
  • System sends a daily email report summarizing:
    • Emails processed
    • Responses sent
    • Resumes generated
    • Any errors or edge cases

This keeps the system understandable and auditable.

UX Considerations (Intentionally Minimal)

This is not a UX-heavy project by design.

  • No user-facing interface
  • No visual dashboards
  • No complex interaction models

The "UX" is:

  • Fewer interruptions
  • Less manual work
  • Clear daily summaries
  • Confidence that nothing is missed
Product Impact
  • Reduced daily email handling time dramatically
  • Eliminated repetitive resume editing
  • Improved consistency and quality of recruiter responses
  • Freed cognitive space for actual interviews and preparation
Future Potential

JobBot intentionally stops short of becoming a full product.

Next steps (if commercialized):

  • Custom web interface
  • Multi-user support
  • Advanced analytics
  • Recruiter response optimization
  • Paid tiers

For now, it functions as:

  • A working MVP
  • A proof of concept
  • A live experiment gathering real-world feedback
